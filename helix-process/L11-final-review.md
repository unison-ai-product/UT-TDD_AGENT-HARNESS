# L11 総合レビュー・ユーザー検証・要件巻き取り

## 工程の位置づけ

| 項目 | 内容 |
|---|---|
| 区分 | V字 右腕（検証フェーズ） |
| 入力 | L10・L1 要求定義・L3 要件定義 |
| 出力 | L12 への入力 |
| 対応する設計 | L1 要求定義・L3 要件定義（最終突合） |

## この工程の PLAN

PLAN は機能（ドキュメント）単位で起票し、工程表（作成手順＋進捗）と実装計画を内蔵する。

### `L11-総合レビューplan`
- 全工程成果物の整合レビュー

### `L11-ユーザー検証plan`
- ユーザー視点での検証

### `L11-要件巻き取りplan`
- 未充足要件・変更要件の巻き取り

## PLAN が内蔵するもの

- **工程表**: そのドキュメントを完成させる手順（例: 参考調査の Web 検索 → 既存資料・ヒアリング整理 → ドラフト → TL レビュー → 確定）と各手順の進捗
- **実装計画**: 記載項目をどう埋めるかの計画
