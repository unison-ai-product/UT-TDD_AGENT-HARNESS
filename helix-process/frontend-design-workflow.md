# フロントデザインワークフロー（UX / ビジュアルデザイン）

## 概要

L10 フロントデザインUX・ビジネスデザイン磨き上げ工程の専門ワークフロー。ビジュアルデザインとデザインシステムで UX を磨き上げる。HELIX の FE 弱点を補うため、UX ビジュアルデザインを専門化する。

## 位置づけ

- Vモデル L10 工程（フロントデザインUX・ビジネスデザイン磨き上げ）の専門化。
- L2 画面設計（ワイヤーフレーム）の右腕ペア。L2 で設計した画面を L10 でビジュアル・UX として磨き上げる。
- 成果物は `L10-ux-refinement.md` に対応する。

## 業界標準プロセス

UX デザインプロセス（ユーザー共感 → 問題定義 → ビジュアルデザイン → デザインハンドオフ → リリース検査）に沿い、デザインシステム / デザイントークンで一貫性を担保する。

- **デザイントークン**: 色・タイポグラフィ・スペーシングを変数化し、ハードコーディングを避けて Single source of truth とする
- **アクセシビリティ（a11y）**: 色コントラスト・フォントサイズ・代替テキスト・地域別要件

## フロー

1. **ビジュアルデザイン**: 色・タイポグラフィ・レイアウトの確定
2. **デザインシステム / デザイントークン**: スタイルを変数化し一元管理（SSOT）
3. **アクセシビリティ（a11y）**: コントラスト・フォント・代替テキストの確認
4. **ビジュアル回帰確認**: 意図しない見た目の変化を検出
5. **UX 磨き上げ**: 操作性・体験の改善
6. **ビジネスデザイン磨き上げ**: ビジネス価値の最適化

## HELIX 検証観点（FE detector）

| detector | 検証 | 現状 |
|---|---|---|
| `design-token-drift` | デザイントークンの定義と実装の乖離 | 定義先行・未実装（本モードで実装目標を明確化） |
| `a11y-regression` | アクセシビリティの後退 | 同上 |
| `visual-regression` | ビジュアルの意図しない変化 | 同上 |

## Forward 接続

磨き上げたデザインは L10 の成果物として L11 総合レビュー・ユーザー検証へ渡す。デザイントークンは実装の Single source of truth として L7 実装と同期する。
