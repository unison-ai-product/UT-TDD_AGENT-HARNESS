# Incident HELIX ワークフロー

## 概要

Incident は、本番運用中に発生した障害・インシデントに緊急即応するモード。開発中の割り込み（interrupt / IIP・CC）や計画的な運用検証（L13 / L14）とは別に、本番稼働中の問題へ hotfix で即応し、収束後に恒久対策を Vモデル体系へ昇華する。Vモデルの弱点である「本番運用中の緊急対応」を埋める。

## 位置づけ（似た機構との違い）

| 機構 | 対象 |
|---|---|
| interrupt（IIP / CC） | 開発中（sprint 中）の設計ギャップ・要件変更の割り込み |
| L13 / L14（運用検証・改善） | 計画的な運用・改善 |
| Discovery post-deploy trigger | 本番後の問題を「検証候補化」する（検出止まり） |
| **Incident** | **本番稼働中の障害への緊急即応（hotfix）と恒久対策への昇華** |

## 入口判定

| 状況 | Incident に入れる理由 |
|---|---|
| 本番障害（サービス停止・重大バグ） | hotfix で即応する |
| SLO / KPI の重大逸脱 | 緊急に原因を特定し対処する |
| セキュリティインシデント | 封じ込めと暫定対処を急ぐ |

検出は Discovery の post-deploy trigger や監視アラートから接続する。

## 基本フロー

```
検出 → トリアージ（影響・緊急度）→ 緊急修正（hotfix）→ 即リリース → 収束確認 → 事後昇華
```

## PLAN kind

| kind | 用途 |
|---|---|
| `troubleshoot` | 障害対応（原因究明と暫定対処） |
| `recovery` | 復旧（復旧手順・ロールバック） |

## Vモデル体系への昇華

緊急対応（暫定）で収束させた後、恒久対策を Forward（Vモデル L0–L14）へ昇華する。hotfix を打ちっぱなしにせず、必ず正式体系に戻す。

```
hotfix で暫定収束
   ↓ ドキュメント化（Reverse fullback を活用）
対応内容（原因・暫定対処・影響）を文書化
   ↓ 恒久対策を Vモデルへ
L1 要求定義 / L3 要件定義 / L4–L6 設計・実装で恒久対策を正式化
   ↓ 学びを運用へ
postmortem を L14 運用検証へフィードバック
```

### 昇華先の対応

| 事後に起こす内容 | 昇華先 |
|---|---|
| 暫定対処の恒久化（要件・設計） | L1 / L3 / L4–L6 |
| 再発防止のテスト | L8 / L9 |
| postmortem・運用学習 | L14 運用検証 |

これにより、緊急の hotfix は速く打ちつつ、恒久対策は Vモデルのドキュメント体系・トレーサビリティに収束する。
